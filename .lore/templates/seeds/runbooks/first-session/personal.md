# First Session Setup: Personal

> **Coming soon.** This runbook is planned for a future release.

**Profile:** Individuals using Lore for personal knowledge management, task tracking, research, and daily workflow — no cloud infrastructure, minimal CLI auth complexity, no enterprise org context.

In the meantime, [knowledge-worker.md](knowledge-worker.md) covers the universal phases: identity setup, service mapping, semantic search, and repo linking. Skip or simplify the phases that don't apply (cloud CLI, enterprise secret stores, org wiki discovery).

**What will differ:**
- Minimal CLI auth — typically a single GitHub account at most
- Secret store optional if no sensitive API keys are in scope
- Service mapping focused on personal tools: notes apps, cloud storage, communication
- Work context from personal goals and projects, not org OKRs
- Simpler model configuration — single tier may be sufficient
- No org wiki — public docs and personal notes are the knowledge sources
