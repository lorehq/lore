# Knowledge

Reference material about your environment, systems, and operations. Loaded on-demand — not injected every session.

This directory is built by the agent as it learns — not pre-authored upfront. Don't create subdirectory structure manually before running first-session setup. Let the agent populate it, then use the knowledge defrag runbook to organize by content.

## Structure

- **Environment** — systems, architecture, accounts, integrations, operations
- **Runbooks** — multi-step operational procedures
- **`local/`** — local-only notes, gitignored (never committed)
