# Skills Registry

| Skill | Domain | Type | Description |
|-------|--------|------|-------------|
| lore-capture | Orchestrator | command | Session-scoped knowledge capture — review work, create skills, update registries |
| lore-consolidate | Orchestrator | command | Deep repo-wide health check — stale items, overlap, duplication |
| lore-create-agent | Orchestrator | command | Create a domain agent when skills cluster around a specific tool |
| lore-create-brainstorm | Orchestrator | command | Create a brainstorm folder — always standalone, never nested |
| lore-create-plan | Orchestrator | command | Create a plan folder with frontmatter and validation |
| lore-create-roadmap | Orchestrator | command | Create a roadmap folder with frontmatter and validation |
| lore-create-skill | Orchestrator | command | Create a new skill when an operation required non-obvious knowledge |
| lore-create-todo-with-capture | Orchestrator | command | Append a capture checkpoint and optimize task lists for parallelism |
| lore-link | Orchestrator | command | Link and unlink work repos to the Lore hub |
| lore-status | Orchestrator | command | Show Lore instance health — version, hooks, skills, agents, active work |
| lore-ui | Orchestrator | command | Start, stop, or inspect the local docs UI. Prefers Docker and falls back to local mkdocs. |
| lore-update | Orchestrator | command | Update Lore framework files to the latest version |
| mcp-stdio-content-length-framing | MCP | — | Cursor MCP uses newline-delimited JSON over stdio, not Content-Length framing |
| node-macos-stdin-fd | Orchestrator | — | macOS pitfalls for Node.js subprocess stdin delivery and tmp dir symlinks |
| platform-command-collisions | Orchestrator | — | Avoid naming skills that collide with platform built-in slash commands |
| pymdownx-extension-order | Orchestrator | — | pymdownx.tilde must be listed before pymdownx.tasklist for strikethrough inside checkboxes |
