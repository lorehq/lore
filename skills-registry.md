# Skills Registry

| Skill | Type | Description |
|-------|------|-------------|
| lore-capture | command | Session-scoped knowledge capture — review work, create skills, update registries |
| lore-consolidate | command | Deep repo-wide health check — stale items, overlap, duplication |
| lore-create-agent | command | Create an operator agent for recurring delegation patterns |
| lore-create-brainstorm | command | Create a brainstorm folder — always standalone, never nested |
| lore-create-plan | command | Create a plan folder with frontmatter and validation |
| lore-create-roadmap | command | Create a roadmap folder with frontmatter and validation |
| lore-create-skill | command | Create a new skill when an operation required non-obvious knowledge |
| lore-create-todo-with-capture | command | Append a capture checkpoint and optimize task lists for parallelism |
| lore-link | command | Link and unlink work repos to the Lore hub |
| lore-status | command | Show Lore instance health — version, hooks, skills, agents, active work |
| lore-ui | command | Start, stop, or inspect the local docs UI. Prefers Docker and falls back to local mkdocs. |
| lore-update | command | Update Lore framework files to the latest version |
| mcp-stdio-content-length-framing | — | Cursor MCP uses newline-delimited JSON over stdio, not Content-Length framing |
| node-macos-stdin-fd | — | macOS pitfalls for Node.js subprocess stdin delivery and tmp dir symlinks |
| platform-command-collisions | — | Avoid naming skills that collide with platform built-in slash commands |
| pymdownx-extension-order | — | pymdownx.tilde must be listed before pymdownx.tasklist for strikethrough inside checkboxes |
